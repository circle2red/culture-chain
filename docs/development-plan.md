# 开发计划

## 项目概览

**项目名称**：CultureChain
**目标**：3 个月内上线 MVP，支持画作、书籍、影视、音乐四类作品的铸造与交易
**团队假设**：1 全栈 + 1 合约工程师（或独立开发者）

---

## 阶段划分

```
Phase 0: 基础设施搭建     (Week 1-2)
Phase 1: 合约开发与测试   (Week 2-4)
Phase 2: 核心前端页面     (Week 3-6)
Phase 3: 交易市场功能     (Week 5-8)
Phase 4: 店铺系统         (Week 7-9)
Phase 5: 测试网集成测试   (Week 9-10)
Phase 6: 安全审计与优化   (Week 10-11)
Phase 7: 主网上线         (Week 12)
```

---

## Phase 0：基础设施搭建（Week 1-2）

**目标**：搭好 Monorepo，所有工程师能在同一环境开发

- [ ] 初始化 Turborepo + pnpm workspace
- [ ] 创建 `apps/web`：Next.js 14 + Tailwind CSS + shadcn/ui

- [ ] 配置 ESLint + Prettier + Husky + lint-staged

- [ ] 配置 CI（GitHub Actions）：lint + test + build
- [ ] 编写 `.env.example`

**交付物**：
- `pnpm dev` 能启动前端开发服务器
- `pnpm contracts:test` 能运行空测试套件

## Phase 1：智能合约开发（Week 2-4）

**目标**：完成所有核心合约，100% 测试覆盖

### 合约开发顺序

#### Week 2：基础合约
- [ ] `CultureNFT.sol`
  - ERC-1155 + ERC-2981 继承
  - `mint` / `mintBatch` / `burn` 函数
  - 版税设置（每个 tokenId 独立版税率，上限 10%）
  - 供应量控制
- [ ] 单元测试：铸造、转移、版税查询

#### Week 3：市场合约
- [ ] `Marketplace.sol`
  - 上架 / 下架逻辑
  - 直购功能（MATIC 支付）
  - 报价 / 接受报价
  - 平台手续费（可配置，默认 2.5%）
  - 自动版税分配
- [ ] 单元测试：完整购买流程、版税分配计算、边界条件

#### Week 4：辅助合约 + 集成测试
- [ ] `ShopRegistry.sol`：店铺注册与元数据管理
- [ ] 集成测试：完整用户旅程（注册店铺 → 铸造 → 上架 → 购买 → 版税到账）
- [ ] Slither 静态安全分析，修复所有 HIGH/MEDIUM 问题
- [ ] 部署到 Mumbai 测试网，记录合约地址

**交付物**：
- 所有合约通过测试，覆盖率 > 90%
- Mumbai 测试网合约地址文档
- `packages/sdk` 中的 TypeChain 类型生成

## 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| Polygon 网络拥堵 | 低 | 中 | 动态 Gas 估算，用户可自行调整 |
| IPFS 节点不稳定 | 中 | 高 | 同时 Pin 到 Pinata + Arweave |
| 合约漏洞 | 低 | 极高 | 充分测试 + 安全审计 + 紧急暂停机制 |
| 监管政策变化 | 中 | 高 | 保留 KYC 接口预留，内容合规审核 |
| 新用户 Web3 门槛 | 高 | 中 | AA 账户抽象（Post-MVP）+ 详细引导 |
